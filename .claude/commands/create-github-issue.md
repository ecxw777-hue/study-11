# GitHub 이슈 생성 커스텀 커맨드 (Create GitHub Issue Custom Command)

## 사용 방법

이 커스텀 커맨드에 생성하고 싶은 이슈 내용을 입력하면, 코드베이스를 분석하여 체계적인 GitHub 이슈를 `gh issue create` 명령으로 자동 생성합니다.

## 프롬프트

다음 내용을 기반으로 GitHub 이슈를 생성해주세요:

**이슈 내용:** $ARGUMENT

아래 절차를 순서대로 수행하세요:

---

### 1단계: 코드베이스 분석

이슈를 작성하기 전에 반드시 현재 코드베이스를 분석하세요:

- 이슈와 관련된 파일, 컴포넌트, 데이터 구조를 파악
- 기존 구현 상태를 확인 (이미 구현된 부분 / 미구현 부분)
- 영향받는 파일 경로를 구체적으로 식별

### 2단계: 이슈 분류

다음 중 적절한 타입을 판단하세요:

| 라벨 | 설명 |
|------|------|
| `feature` | 새로운 기능 추가 |
| `bug` | 버그 수정 |
| `enhancement` | 기존 기능 개선 |
| `refactor` | 코드 구조 개선 (동작 변경 없음) |
| `docs` | 문서 작성/수정 |
| `chore` | 설정, 의존성, CI/CD 등 유지보수 |

### 3단계: 이슈 작성 및 생성

분석 결과를 바탕으로 `gh issue create` 명령을 사용하여 이슈를 생성하세요.

이슈 본문은 아래 템플릿을 따르되, 이슈 타입에 맞게 불필요한 섹션은 제거하세요:

```markdown
## 개요
[이슈의 핵심을 1-2문장으로 요약]

## 현재 상태
[코드베이스 분석 결과 — 현재 어떤 상태인지]

## 목표
[이 이슈가 완료되면 달라지는 점]

## 작업 내용
- [ ] 구체적인 작업 항목 1
- [ ] 구체적인 작업 항목 2
- [ ] 구체적인 작업 항목 3

## 관련 파일
- `path/to/file1.tsx` — 설명
- `path/to/file2.ts` — 설명

## 참고 사항
[기술적 제약, 주의점, 또는 관련 이슈 번호]
```

### 이슈 제목 규칙

- 타입 접두사 포함: `[feature]`, `[bug]`, `[enhancement]` 등
- 한국어로 작성
- 핵심 내용을 간결하게 (50자 이내)
- 예시: `[feature] 도시 상세 페이지에 좋아요 기능 추가`

### 생성 명령

```bash
gh issue create --title "제목" --body "본문" --label "라벨"
```

- `--label`이 저장소에 없을 경우 라벨 없이 생성
- 생성 후 이슈 URL을 사용자에게 반환

---

**추가 지침**

- 하나의 이슈에는 하나의 주제만 포함 (범위가 크면 여러 이슈로 분리 제안)
- 작업 내용의 각 항목은 구체적이고 검증 가능해야 함
- 코드베이스에서 확인한 실제 파일 경로를 사용 (추측하지 않기)
- 이슈 생성 전 제목과 본문을 사용자에게 미리 보여주고 확인 후 생성
